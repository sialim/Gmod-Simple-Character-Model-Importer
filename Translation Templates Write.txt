Simple Dynamic Model Importer - Import Any Dynamic Model into Gmod (Utility)

[b]You can download any MMD model from Aplaybox, Bowroll, Deviantart etc... and import it into Garry's Mod with this addon without restaring your game. [/b] This addon does real-time import of any standard PMX (MikumikuDance) formatted, dynamic (character) models into Garry's Mod. They will be imported into Gmod as PM & NPCs.

[h1]Introduction[/h1]
Have you noticed that, unlike most modern games, you have to port a character into Source Engine format before you can actually use it in Garry's Mod? That process is slow, repetitive, and difficult, which limits the availability of character model for the community. Well, that is no longer the case. Here we introduce "Simple Dynamic Model Importer" to fully address this issue.

You need to use the provided opensourced importer tool to convert supported model files into Garry's Mod-ready mesh/material data, then spawn them in-game with the Toolgun. Download the tool here: [url=https://github.com/SheepyLord/Gmod-Simple-Static-Model-Importer/releases/tag/Release4] GitHub [/url]

This addon is for player and creator who wants to bring [b]custom character models[/b] into Garry's Mod without going through the full Source engine model porting and compile pipeline. This addon currently only supports models in standard .pmx (Polygon Model eXtended for MikuMikuDance) format.

[h1]Features[/h1]
- Import almost any character model as playermodel and NPC into Garry's Mod.
- Supports Facial Expression, VRD, Physics and PBR material (RTX-Remix only).
- Supports characters with normals, UVs, and per-material textures.
- Spawn imported models in-game with the Toolgun or Spawnmenu.
- Models are server-sided and shareable between clients.
- Select which parts of the model should be imported.
- The import is customizable with Advanced Options.
- Save / Load between sessions
- Export your imported models as Garry's Mod addons and release it to the Steam workshop.

[h2]How to use[/h2]
[list=1]
[*]Subscribe to this Workshop item.
[*]Download the open-source Python importer tool from GitHub.
[*]Enter a valid [b]Display Name[/b] for your model.
[*]Run the open-source Python importer tool from GitHub.
[*]Launch Garry's Mod.
[*]Open the Toolgun and go to [b]Model Importer -> Dynamic Model Importer[/b].

[h3]Known Issues[/h3]
- Crash or fail on complicated model, especially when importing / loading models with more than 192k vertices.
- The program is slow, it needs ~10 minutes to port a model into Gmod.
- Slight offset between the autogenerated physics / hitbox model and the acutal mesh.
- Issues related to textures and jigglebones for some models.
- Character Models in other format (.fbx and .uemodels models) must be converted into standard .pmx model for the importer to work. If you character can play dances in Mikumikudance, it will work with this tool.
